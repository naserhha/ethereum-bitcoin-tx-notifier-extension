Contributing Guide

We welcome contributions from the open source community! Whether you are a developer, translator, designer, or security researcher, your input helps make this extension better for everyone.

  • 🐛 Report Issues: Found a bug or have a feature request? Please open an issue on GitHub with clear details and steps to reproduce.
  • 🔀 Submit Pull Requests: Fork the repository, create a feature branch, and submit a pull request. Please describe your changes and reference related issues.
  • 📝 Coding Standards: Follow clean code practices, use descriptive commit messages, and keep code modular and well-documented. See existing code for style reference.
  • 🌍 Add/Improve Translations: Help us support more languages! Update translations.js and submit your translation via pull request.
  • 🔒 Security: If you discover a vulnerability, please report it responsibly via a private GitHub issue or email. Do not disclose security issues publicly until they are resolved.
  • 🤝 Community Guidelines: Be respectful, constructive, and inclusive. All contributions are reviewed for quality and alignment with project goals.
